# Dockerfile

---

- 도커파일은 도커 이미지를 내려받아 패키지를 설치하고, 다시 배포하는 일련의 과정을 자동화 할 수 있게 해주는 일련의 명령어로 구성된 파일이다.   

  

##### Commands

---

- FROM : 도커파일로 생성할 이미지의 베이스가 되는 이미지. Dockerfile 작성시 반드시 한번 이상 입력해야 함
- LABEL : 이미지에 메타데이터 추가. 작성자 등 저장 가능. <키 : 값> 의 형태. 이후에 이 메타데이터를 활용하여 원하는 조건의 이미지를 필터링 할 수 있음
- RUN : 컨테이너 내부에서 명령어 실행. 별도의 입력을 받을 수 없기 때문에 입력을 받아야 하는 명령어가 있으면 오류 발생
- ADD : Dockerfile이 위치한 디렉토리에서 파일을 가져와 이미지에 추가. 외부 URL 및 tar에서도 파일 추가 가능
    - cf) COPY : ADD의 역할에서 로컬 파일 추가만 가능
- WORKDIR : 쉘의 cd와 같은 역할
- EXPOSE : 도커 이미지에서 노출할 포트 번호 설정
- ENV : 도커파일에 환경변수 지정. 해당 환경변수는 이미지에도 적용됨.
- VOLUME : 호스트와 공유할 컨테이너 내부 디렉토리 설정
- ARG : docker build에서 입력 받을 argument 지정. 함수 생성시 argument 지정과 같은 원리. dafult 값 설정 가능.
- USER : 사용자 권한 설정. USER 밑에 있는 명령어들은 해당 유저의 권한으로 실행됨.
- ONBUILD : 빌드된 이미지를 기반으로 하는 다른 이미지가 도커파일로 생성될 때 실행할 명령어를 추가함.
- SHELL : 도커파일에서 기본적으로 사용하는 쉘은 리눅스와 윈도우에서 각각 다름. 이를 직접 지정하고 싶을 때 사용
    - 리눅스 :  /bin/sh -c
    - 윈도우 : cmd /S /C

- CMD : 컨테이너가 실행될 때 마다 실행할 명령어 설정. 한번만 사용 가능
- ENTRYPOINT : CMD와 비슷한 역할. 단, ENTRYPOINT와 CMD가 같이 사용되었을 경우에는 CMD는 ENTRYPOINT의 인자로서의 역할만 수행하게 됨.
    - 모든 dockerfile은 반드시 CMD 혹은 ENTRYPOINT가 적어도 하나는 설정되어야 함
    - CMD와 ENTRYPOINT는 JSON 배열 형태로 입력해야 그대로 실행됨
        - JSON 배열 형태로 입력하지 않으면 앞에 /bin/sh -c 가 붙은 상태로 실행됨  
        
          

##### Docker Context

---

: Dockerfile이 위치해 있는 디렉토리.

- Dockerfile build시에 가장 먼저 읽는 것이 도커 컨텍스트의 정보

- 따라서 Dockerfile이 위치한 디렉토리는 불필요한 파일을 두지 않는 것이 중요

- .dockerignore로 도커 컨텍스트에서 읽지 않을 파일을 정할 수 있음

  

##### docker build

---

- 생성한 Dockerfile을 바탕으로 이미지를 생성하기 위한 도커 커맨드

```bash
docker build -t <IMAGE_NAME> <DOCKERFILE_DIRECTORY>
```

- Arguments
    - -f : 읽어들일 도커파일 이름 지정
    - -t : 생성될 이미지 이름 지정
    - --no-cache : 캐시 사용하지 않음
    - --cache-from : 캐시를 받아올 이미지를 지정
    - --build-arg : 도커파일의 ARG에 값 입력
- Cache
    - 도커파일 빌드 시에 이전에 빌드했던 명령과 똑같은 줄의 명령이 있을 경우, 새로 수행하는 대신에 캐시를 불러와서 더 신속하게 빌드가 가능함.
    - 그러나 업데이트 등이 필요해서 캐시 파일을 사용하면 안되는 경우 (ex. RUN git clone) 캐시를 사용하지 않게 하거나, 캐시를 받아올 이미지를 직접 지정해야 함
- Multi-stage build
    - 한번의 빌드 안에 여러 번의 FROM 을 활용 하는 것
    - 이미지에서 원하는 최종 실행 파일만 가져와서 이미지의 크기를 줄이기 위해 사용
- Image Layer
    - 도커파일 빌드는 한 줄마다 새로운 이미지 레이어를 생성하는 작업임.
    - 도커파일 빌드 과정에서 용량을 생성하고 지워야 하는 부분이 있다면, 이 부분의 명령어를 한 줄 안에 동시에 시행해 주는 것이 좋음

        → 도커파일은 빌드 과정에서 한줄에 하나의 이미지를 생성하기 때문에, 다른 줄에서 파일을 지우더라도 파일이 남아있는 이미지 레이어가 남아있어 용량을 차지하기 때문

    - 다른 사람이 만든 도커파일에서 이미지 레이어 용량을 줄이고 싶은 경우 export 하고 import 하면 이미지 레이어가 하나가 됨
        - 대신 각종 이미지 설정은 잃어버리게 됨
